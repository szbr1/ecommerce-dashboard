## üöÄ Getting Started

### Prerequisites

Ensure you have the following installed:

```bash
Node.js >= 18.0.0
PostgreSQL >= 14.0
pnpm >= 8.0.0 (recommended)
```

### Installation

1Ô∏è‚É£ **Clone the repository**

```bash
git clone https://github.com/szbr1/ecommerce-dashboard.git
cd ecommerce-dashboard
```

2Ô∏è‚É£ **Install dependencies**

```bash
# Install all dependencies (root, client, and server)
pnpm install

# Or install separately
cd client && pnpm install
cd ../server && pnpm install
```

3Ô∏è‚É£ **Environment Setup**

<details>
<summary><b>Server Configuration (.env)</b></summary>

Navigate to `server/` directory and create `.env`:

```env
# PostgreSQL Database
DATABASE_URL="postgresql://postgres:your_password@localhost:5432/board"

# Cloudinary Configuration
CLOUD_NAME=your_cloudinary_name
CLOUD_KEY=your_cloudinary_key
CLOUD_SECRET=your_cloudinary_secret

# GitHub OAuth (Optional - for future auth implementation)
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
```

**üìù Getting Credentials:**
- **PostgreSQL**: Use your local installation credentials
- **Cloudinary**: Free signup at [cloudinary.com](https://cloudinary.com)
- **GitHub OAuth**: Create app at [GitHub Developer Settings](https://github.com/settings/developers)

</details>

<details>
<summary><b>Client Configuration (.env.local)</b></summary>

Navigate to `client/` directory and create `.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:4000
```

</details>

4Ô∏è‚É£ **Database Setup**

```bash
cd server

# Run migrations
npx prisma migrate dev --name init

# Generate Prisma Client
npx prisma generate

# Seed database with sample data
pnpm seed
```

5Ô∏è‚É£ **Run the Application**

**Terminal 1 - Backend Server:**
```bash
cd server
pnpm dev
# Server runs at http://localhost:4000
```

**Terminal 2 - Frontend Client:**
```bash
cd client
pnpm dev
# Client runs at http://localhost:3000
```

6Ô∏è‚É£ **Access the Dashboard**

Open your browser and navigate to:
```
http://localhost:3000
```

## üéØ Key Technical Highlights

### üîÑ State Management with RTK Query

Efficient data fetching with automatic caching, refetching, and optimistic updates:

```typescript
// Automatic caching and real-time updates
const { data: products } = useGetProductsQuery();
const [updateProduct] = useUpdateProductMutation();
```

### üì§ Image Upload Pipeline

Seamless image handling from client to cloud:

```typescript
Client (Drag & Drop) ‚Üí Multer (Processing) ‚Üí Cloudinary (Storage) ‚Üí Database (URL)
```

### üîí Type Safety with Prisma

Auto-generated types for database operations:

```typescript
// Fully typed database queries
const products = await prisma.product.findMany({
  include: { category: true, reviews: true }
});
```
